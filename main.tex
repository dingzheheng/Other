\section{游戏引擎是什么?}
游戏引擎这个术语起源了第一人称游戏DOOM.DOOM使用了良好定义的核心软件组件(例如三维图形渲染引擎,冲突检测,音频系统)划分,艺术assets,游戏世界,玩家规则.分离的价值在游戏开始授权和重建一个新的产品,通过创建新的艺术,世界布局,武器,角色,车辆和游戏规则,只需要一小点改变对引擎软件开始实现.

游戏和他的引擎之间的界限总是模糊不清.


\section{引擎风格(类型)的不同}
游戏引擎的风格是特定的.一个用于设计两人对打拳击游戏的引擎与MMOG引擎或者FPS引擎或者RTS引擎是非常不同的.然而,这里也有很多重叠的部分,例如所有的3D引擎,不管他的风格,都需要一些底层的用户接口,用于处理joypad,键盘,鼠标,3D Mesh渲染,警告显示(不同字体的文本渲染,强大的音频系统).

让我们来看看最常用的游戏引擎风格,并探索技术范例,每个引擎风格的部分需求.

\subsection{FPS}
FPS关注的典型技术:\\
1. 高效的大型3D虚拟世界的渲染.\\
2. 摄像机控制/瞄准机制的反应.\\
3. 高保真的玩家的虚拟装备和武器动画.\\
4, 大范围的强力手持武器\\

\subsection{格斗游戏}
关注的技术:\\
1. 一个丰富的战斗动画集.\\
2. 精确打击判定.\\
3. 用户输入系统有能力检测复杂按钮和手柄结合.\\
4. 群众,相应的静态背景.

\subsection{竞速游戏}
技术关注:\\
1. 不同的诀窍,当渲染距离背景元素,例如使用二维卡片作为树,山,小山.\\
2. 跑道总是切断成相对简单的二维区域,叫做区间.这些数据结构用于优化渲染和可视化确定,帮助ai和路径寻找,用于非人类控制汽车,解决其他技术问题.

\subsection{RTS}
技术要点:\\
1. 每个单位相对低资源,这样就可以在屏幕中同时显示大量的单位\\
2. 玩家通常在地形上创建建筑用于部署他们的力量\\
3. 典型的使用单击或者基于区域中单位的选择,加上菜单或者工具栏包含命令,装备,单位类型,建筑类型等.
4. 具有高度的地形通常被使用
